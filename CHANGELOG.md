# Changelog

All notable changes to this project are documented here. The format is based on
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres
to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.0.2] - 2026-07-01

### Fixed
- Path resolution on **Windows PowerShell 5.1**: single-file (and pipeline)
  invocations previously reported "No PDF files to check" and exited 2, because
  the `process{}` block does not run when a `ValueFromPipeline` parameter is bound
  positionally via `-File`. Rewrote to `Test-Path`-based expansion with an `end{}`
  fallback that behaves identically on 5.1 and 7. No behavior change on 7.

## [0.0.1] - 2026-07-01

Initial release.

### Added
- `Check-PdfComments.ps1` — report-only pre-filing gate that detects markup /
  review annotations (sticky notes, highlights, strikeouts, stamps, redaction
  marks, etc.) in PDFs and exits non-zero when any are found. Deliberately does
  not flag `/Link` annotations so interactive TOC / cross-reference links survive.
  Pure .NET: inflates Flate-compressed object streams in-process and decodes
  comment text (including UTF-16 with a BOM and backslash line-continuations).
- Windows convenience wrappers: `Check-PdfComments.bat` (drag-and-drop) and
  `Install-ContextMenu.bat` / `Uninstall-ContextMenu.bat` (per-user right-click).
- Dependency-free test suite (`tests/Run-Tests.ps1`) with reproducible fixtures
  generated by `tests/make-fixtures.py`.
- GitHub Actions CI running the suite on Linux (PowerShell 7) and Windows
  (PowerShell 7 and Windows PowerShell 5.1).
