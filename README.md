# pdf-comment-gate

[![tests](https://github.com/lizthegrey/pdf-comment-gate/actions/workflows/ci.yml/badge.svg)](https://github.com/lizthegrey/pdf-comment-gate/actions/workflows/ci.yml)

A dependency-free **pre-filing gate** that detects review comments and markup
annotations left inside a PDF before it's shared or filed. It exists to catch the
common mistake of sending out a PDF with internal sticky-note comments still
embedded — comments that are invisible in a quick preview but trivially
recoverable from the file.

It is **report-only**: it never modifies the PDF. When it finds markup it prints
what and who, and exits non-zero so it can be wired into a checklist, a hook, or a
right-click action. A human removes the comments and re-runs it.

## What it flags (and what it deliberately does not)

Markup / review annotation subtypes:

> `Text`, `FreeText`, `Highlight`, `Underline`, `StrikeOut`, `Squiggly`,
> `Caret`, `Ink`, `Stamp`, `FileAttachment`, `Sound`, `Redact`, `Line`,
> `Square`, `Circle`, `Polygon`, `PolyLine`

It **does not** flag `/Link` annotations. Table-of-contents and cross-reference
hyperlinks are legitimate — some workflows require them to stay active — so links
must not trip the gate.

- **`Redact`** annotations mean redaction rectangles that may **not** be burned in
  yet; apply them before sharing.
- **`Stamp`** annotations are often signature or exhibit stamps — real floating
  annotations, not page content — so the gate surfaces them and lets a human
  decide. Flatten your own stamps before filing.

It handles annotations whether stored uncompressed or inside Flate-compressed
object streams, and decodes comment text in the common encodings (including
UTF-16 with a BOM and strings wrapped across lines with backslash continuations).

## Requirements

**Windows PowerShell 5.1** (ships with Windows) **or PowerShell 7+**. Pure .NET —
no modules, no Acrobat, no Ghostscript. PowerShell 7 also runs on **Linux and
macOS**, so the gate works there too (handy as a CI check or git hook). Only the
`.bat` wrappers and the right-click installer are Windows-only.

## Usage (Windows)

```powershell
.\Check-PdfComments.ps1 .\ready-to-file.pdf          # one file
.\Check-PdfComments.ps1 .\Filings                     # a folder (top level)
.\Check-PdfComments.ps1 .\Filings -Recurse -IncludeMetadata
```

## Usage (Linux / macOS)

Install PowerShell 7 (`pwsh`), then call the script the same way:

```bash
# install pwsh — pick one:
sudo snap install powershell --classic          # Ubuntu/Debian/Fedora via snap
# or a self-contained tarball (no root): see https://aka.ms/powershell-release
brew install --cask powershell                  # macOS

# run the gate
pwsh -File ./Check-PdfComments.ps1 ./ready-to-file.pdf
pwsh -File ./Check-PdfComments.ps1 ./Filings -Recurse
echo "exit code: $?"                             # 0 clean / 1 dirty / 2 error
```

As a **git pre-commit hook** to block committing PDFs that carry comments:

```bash
#!/usr/bin/env bash
# .git/hooks/pre-commit
mapfile -t pdfs < <(git diff --cached --name-only --diff-filter=ACM -- '*.pdf')
[ ${#pdfs[@]} -eq 0 ] && exit 0
pwsh -File /path/to/Check-PdfComments.ps1 "${pdfs[@]}"   # non-zero blocks the commit
```

Exit codes (for hooks / gates):

| code | meaning |
|------|---------|
| `0`  | no markup annotations found |
| `1`  | comments / markup found |
| `2`  | read error, or no PDFs matched |

A `CLEAN` result means "no markup annotations found," not a guarantee the file is
safe: it can't see inside an encrypted PDF, and doesn't report document metadata
unless you pass `-IncludeMetadata`.

**Drag-and-drop:** drop PDFs onto `Check-PdfComments.bat` (keeps the window open).
**Right-click:** run `Install-ContextMenu.bat` (per-user, no admin) to add a
"Check for review comments" entry; on Windows 11 it's under "Show more options."
`Uninstall-ContextMenu.bat` removes it.

## Remediation — remove comments, keep your links

Do **not** use *Sanitize Document* or *Flatten annotations*: both destroy
interactive TOC / cross-reference links. Instead, in Acrobat Pro:

1. **Comment pane** — select all comments, right-click > Delete; or
2. **Remove Hidden Information** — tick **only** "Comments and Markup" (leave
   "Links, Actions and JavaScript" unticked).

Then re-run the gate to confirm `CLEAN`.

## Tests

```powershell
pwsh -File tests/Run-Tests.ps1
```

Fixtures under `tests/fixtures/` are tiny hand-built PDFs regenerated by
`tests/make-fixtures.py`, covering a link-only PDF (must stay CLEAN), an
uncompressed sticky-note comment, and a comment inside a compressed object stream.
